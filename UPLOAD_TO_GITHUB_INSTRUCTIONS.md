# Upload Instructions

1. Create a new GitHub repo named `eonapp-public-showcase`.
2. Keep it separate from the private EONAPP production repo.
3. Upload the contents of this ZIP only.
4. Do not add `.env`, production source, Cloudflare secrets, wallet/private keys, provider keys, Telegram bot tokens, Monetag secrets, Filebase keys, or NOWPayments secrets.
5. Enable GitHub Actions.
6. Confirm the `Public Showcase Check` workflow passes.
7. Review screenshots and README on GitHub mobile and desktop.
8. Link the repo from EONAPP `/trust.html` and Code Showcase only after final review.

Recommended repo description:

> Public-safe EONAPP code showcase: curated snippets, screenshots, architecture notes, trust model, and security boundaries. Full app source remains proprietary.
