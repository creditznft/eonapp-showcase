# Public vs Private Boundary

## Public-safe

- screenshots
- architecture notes
- selected simplified snippets
- trust model docs
- security reporting policy
- sanitized examples

## Private

- full app source
- deployment scripts with secret assumptions
- production Cloudflare functions
- API keys and `.env` files
- wallet private keys
- full monetization internals
- full orchestration logic
- proprietary EON City implementation

## Why this boundary exists

A public showcase should build trust and credibility without giving away the competitive app or exposing users to secret leakage risk.
